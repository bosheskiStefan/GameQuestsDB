# GameQuestsDB
A database for game quests.
My "Database 1" final project in SQL.
This project is about building a database for a video game, which helps organize all the important game-related data—players, NPCs, quests, rewards, and the relationships between them. The goal was to create a structure that reflects the game world while keeping the data clean and easy to manage.
Databases like this are at the core of almost every modern game. They keep track of who the players are, what they've achieved, and how they interact with the world. Without a well-designed database, it’s easy for things to get messy, which can lead to bugs, inconsistent data, or even a bad experience for the players.
This document outlines the process of designing, creating, and testing the database. It starts with an ER diagram to map out the concepts, then moves into the technical details like SQL implementation and sample queries. The result is a reliable database that could be the backbone of any game with rich interactions.
